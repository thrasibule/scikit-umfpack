Release TasksÂ¶

  1. Setup pypi and testpypi accounts if you do not already have ones. Then put the following into your ~/.pypirc:

    [distutils]
    pypi
    testpypi

[pypi]
repository = https://pypi.python.org/pypi
username = your_pypi_name
password = your_pypi_password

[testpypi]
repository = https://testpypi.python.org/pypi
username = your_testpypi_name
password = your_testpypi_password

    Then register your accounts:

    # testpypi
python setup.py register -r https://testpypi.python.org/pypi

# pypi
python setup.py register

    To be able to upload to pypi/testpypi without sending your password in plain text, install twine. An upload is then done simply by:

    # testpypi
twine upload -r testpypi <filenames>

# pypi
twine upload <filenames> # Be careful!

    WARNING: Uploads to pypi are permanent and cannot be updated later! If a wrong file is uploaded, the only fix is to create a new release. Always use testpypi first.

  2. Make sure that pypandoc is installed so that Markdown to ReST conversion of README.md in setup.py works.

  3. Bump version number:

    • in setup.py
    • in docs/conf.py

  4. Regenerate and review the documentation:

    python setup.py build_ext -i build_sphinx
firefox build/sphinx/html/index.html

  5. Install and test the git version that is to be released. In the git repository, do:

    python setup.py install --user
cd ..
nosetests -v scikits.umfpack
cd scikit-umfpack

  6. Create a source distribution tarball, install it and test it:

    python setup.py sdist
# unpack it, cd to it, then:
python setup.py install --user
cd ..
nosetests -v scikits.umfpack

  7. Upload to testpypi and test:

    python setup.py sdist
twine upload -r testpypi dist/scikit-umfpack-<version>.tar.gz

    Note: if the upload fails with This filename has previously been used, you should use a different version., just change the version in setup.py to another value.

    Create a test install using virtualenv:

    cd tmp/

unset PYTHONPATH

virtualenv test-umfpack

cd test-umfpack
source bin/activate

pip install nose numpy scipy
pip install --pre -i https://testpypi.python.org/pypi scikit-umfpack

nosetests -v scikits.umfpack

deactivate

  8. Upload to pypi:

    • Check the version numbers in setup.py and docs/conf.py

    • Do:

      python setup.py sdist
twine upload dist/scikit-umfpack-<version>.tar.gz

    • For testing, see the previous step.

  9. Update gh-pages:

    ./docs/do-gh-pages.sh
git push -f origin gh-pages

  10. If wheels are available for the released version, upload them also using twine.
